Tomcat

是一个免费的、开源的 Servlet 容器，启动方式：

在下载位置，找到 bin 目录下的 startup.bat 文件，双击即可

检测是否链接：http://localhost:8080/


修改端口号

默认的端口号是 8080，若修改，则在下载位置，找到 conf 目录下的 server.xml 配置文件

找到 Connector 标签，修改 port 属性为需要的端口号，之后重启

Tomcat & idea

具体配置和常见问题另见IDEA 创建一个 web 项目_XXSfirtree的博客-CSDN博客

  • src – 存放自己编写的 java 源代码
  • web – 存放 web 工程的资源文件，比如 html、css 等
  • WEB-INF 是受服务器保护的目录，浏览器无法直接访问
  • lib – 手动创建，存放需要的 jar 包，需要配置
  • web.xml 整个 web 工程的描述配置文件，用于配置组件等
  • index.jsp 默认首页

Servlet

  • 是 JavaEE 规范之一（规范就是接口）
  • 是 JavaWeb 三大组件之一：Servlet 程序、Filter 过滤器、Listener 监听器
  • 是运行在服务器上的小程序，接受客户端发送的请求，并响应数据给客户端

手动创建 Servlet

  1. 编写类实现 Servlet 接口，主要是实现 service 方法，处理请求并响应数据
  2. 在 web.xml 文件配置 Servlet 程序的访问地址

在 web.xml 中配置包括两部分：

  • 先通过<servlet></servlet>进行注册，这部分的 name 要求唯一
  • 在通过<servlet-mapping>完成映射

也可以直接在 Servlet 源文件中通过注释配置


生命周期

  1. servlet 构造器方法
  2. init() 方法 初始化
  3. service 处理请求
  4. destory() 方法销毁

第1、2 是在第一次访问的时候创建 servlet 调用

第3 每发一次请求调用一次

第4 在 web 工程停止的时候调用


针对不同的提交方式处理请求

但是一般情况下我们都采用继承 “HttpServlet” 子类实现 Servlet

通过 HttpServlet 实现 Servlet

  1. 编写一个类继承 HttpServlet 类
  2. 根据业务需要重写 doGet 或 doPost 方法
  3. 到 web.xml 中配置 Servlet 程序的访问地址（跟上面配置方式一样）

Servlet 的继承体系


Servlet 内置对象

ServletConfig

是 Servlet 的配置信息类

ServletConfig 是每个 Servlet 创建时对应创建的，它的作用为：

  1. 可以获取 Servlet 程序的别名（servlet-name
  2. 获取初始化参数 init-param
  3. 获取 ServletContext 对象

ServletContext

是一个接口，表示 Servlet 上下文对象，一个 web 工程只有一个 ServletContext 对象

ServletContext 是一个域对象：可以像 Map 一样保存数据的对象


ServletContext 的主要作用：

（1）获取 web.xml 中配置的上下文参数 context-param（整个 web 工程的共用参数）

（2）获取当前的工程路径：/工程路径

（3）获取工程部署后在服务器硬盘上的绝对路径

（4）存取数据

  • setAttribute(“key”, “value”) – 存数据
  • getAttribute() – 取数据
  • removeAttribute() – 删除数据

HttpServletRequest

每次只要有请求进入Tomcat服务器，Tomcat 服务器就会把请求过来的HTTP协议信息解析好封装到Request 对象中，然后传递到 service 方法，我们可通过 HttpServletRequest 对象得到 request 信息

常用方法：

方法 描述
getRequestURI() 获取请求的资源路径
getRequestURL() 获取请求的统一资源定位符 (绝对路径)
getRemoteHost() 获取客户端的ip地址
getHeader() 获取请求头
getParameter( ) 获取请求的参数
getParameterValues() 获取请求的参数（多个）
getMethod() 获取请求的方式
setAttribute(key, value) 设置域数据
getAttribute(key) 获取域数据
getRequestDispatcher( ) 获取请求转发对象（请求转发以’/‘开始）

解决中文乱码的问题：

必须要在获取请求参数之前调用，否则无效


HttpServletResponse

与 HttpServletRequest 很像，每次请求都创造一个 Response 对象，表示所有相应信息

一般用于设置返回值给用户

返回信息给用户的方式：流

流 方法 描述
字节流 getOutputStream() 常用于下载（传递二进制数据）
字符流 getWriter() 常用于回传字符串

两个流只能使用其中之一，否则报错

eg. 返回字符串给用户

返回值字符串中有中文乱码的解决方式：

必须要在获取流对象之前调用，否则无效


HTTP

所谓HTTP协议，就是指客户端和服务器之间通信时，发送的数据，需要遵守的规则，叫HTTP协议

HTTP协议中的数据又叫报文

GET 请求

POST 请求

常见请求头

  • Accept：表示客户端可以接收的数据类型
  • Accpet-Languege：表示客户端可以接收的语言类型
  • User-Agent：表示客户端浏览器的信息
  • Host：表示请求时的服务器ip和端口号

以下都是 get 请求：

  1. a 标签
  2. link 标签引入 css
  3. Script 标签引入 js 文件
  4. img 标签引入图片
  5. iframe 引入 html 页面
  6. 在浏览器地址栏中输入地址后回车

响应


响应码

  • 200 – 表示请求成功
  • 302 – 表示请求重定向
  • 404 – 表示服务器已收到请求，但是要的数据不存在
  • 500 – 表示服务器已收到请求，但是服务器内部错误

MIME 类型说明

MIME 是 HTTP 协议中数据类型，多功能 Internet 邮件扩充服务

MIME 类型的格式是“大类型 / 小类型”，并与某一种文件的扩展名相对应

请求的转发

forward() 转发的特点

  • 转发后地址栏没有变化
  • 是一次请求，共享 request 域中的数据
  • 可以转发到 WEB-INF 下的文件
  • 不能转发到 web 工程外的地址

<base> 标签设置页面相对路径参考的地址

/ 的含义

（1）在 web 中，‘/’ 是一种绝对路径，表示 http://ip:port

<a href="/">斜杠</a>

（2）在服务器中，‘/’ 是http://ip:port/工程路径

  • <url-pattern>/servlet1</url-pattern>
  • rervletContext.getRealPath(“/“)
  • request.getRequestDispatcher(“/“)

PS：response.sendRediect(“/“); – 表示把斜杠发送给浏览器解析，得到的是 http://ip:port
