！！组件

11

一个完整的 Java Web 应用程序通常是由多种组件构成的，一般由表示层组件、控制层组件、业务逻辑层组件以及数据访问层（或持久层）组件组成：

  • 表示层组件：一般由 HTML 和 JSP 页面组成
  • 控制层组件：一般是 Servlrt、过滤器、监听器
  • 业务逻辑层：一般是 JavaBean 或 EJB
  • 持久层：一般是 JDBC、Hibernate

jsp 元素分为三类型：基本元素、指令元素、动作元素

指令元素

JSP 指令是被 Web 服务器解释并被执行的。通过指令元素可以使服务器按照指令的设置执行动作或设置在整个 JSP 页面范围内有效的属性。在一条指令中可以设置多个属性

JSP指令包括：page 指令、include 指令和 taglib 指令

（1）page 指令：定义整个页面的全局属性

（2）include 指令：用于包含一个文本或代码的文件

（3）taglib 指令：用来引用自定义的标签或第三方标签库


JSP指令的语法格式:

<%@ 指令名称 属性1="属性值1" 属性2="属性值2" .. 属性n="属性值n" %>


1. page 指令

作用域为他所在的 jsp 文件和其包含的文件

属性 说明
language 指定用到的脚本语言，默认是java
import 用于导入 Java 包或 Java 类
pageEncoding 指定页面所用编码，默认与 contentType 相同
extends JSP 转换成 Servlet 后继承的类
session 指定该页面是否参与到 HTTP 会话中
buffer 设置 out 对象缓冲区大小
autoflush 设置是否自动刷新缓冲区

contentType 设计响应 JSP 页面的 MIME 类型和字符编码
isErrorPage 设置是否是一个错误处理页面
isELIgnord 设置是否忽略正则表达式

！！！include 指令和 include 动作的区别：

两者的运行机制不同，他们所实现的两种包含，程序的执行性质完全不同。

include 指令是静态包含，先合成再编译；include 动作标签是动态包含，先编译再合成。

对于静态包含不能传递参数，而对于动态包含可以在两个文件之间传递参数


2. jsp:forward

转发页面，当前页面不执行

<jsp:forward page="文件的名字" />


33

！！！使用response对象的sendRedirect() 方法和转发<jsp:forward page=”url”> 的区别：

（1）<jsp:forward>只能在本网站进行跳转，而response.sendRedirect可以跳转到任何一个地址的页面

（2）<jsp:forward>带 request 中的信息跳转，sendRedirect不带 request 中的信息跳转

（3）使用<jsp:forward>时浏览器的地址栏不发生变化，而sendRedirect会显示跳转后的URL地址

！！！get 和 post 的区别：

（1）get 的参数会显示在浏览器地址栏中，而 post 的参数不会显示在浏览器地址栏中

（2）使用 post 提交的页面在点击【刷新】按钮的时候浏览器一般会提示“是否重新提交”，而 get 则不会

（3）用 get 的页面可以被搜索引擎抓取，而用 post 的则不可以

（4）用 post 可以提交的数据量非常大，而用 get 可以提交的数据量则非常小(2k)，受限于网页地址的长度

（5）用 post 可以进行文件的提交，而用 get 则不可以

结果集结构 ResultSet

java.sql.ResultSet 接口类似于一个数据表，通过该接口的实例可以检索结果集，以及对应数据表的相关信息。

ResultSet 实例是通过执行查询数据库的语句生成的


ResultSet 实例具有指向其当前数据行的指针。最初，指针指向第一行记录的前方，通过 next() 方法指向下一行，没有下一行时将返回 false

方法名称 功能描述
first() 指针移到第一行
last() 指针移到最后一行
previous() 指针移到上一行
next() 指针移到下一行
getRow() 查看当前行的索引编号
findColumn() 按指定列名查看列名的索引编号，没有则抛出异常
close() 释放资源

PS：索引编号从 1 开始

JavaBean

设计规则

55

（1）JavaBean 是一个公共类

（2）JavaBean 类具有一个公共无参的构造方法

（3）JavaBean 所有的属性定义为私有的

（4）在 JavaBean 中，需要对每个属性都提供两个公共方法：getXxx() 和 getXxx()

（5）定义 JavaBean 时，通常放在一个命名的包下


基于JSP动作标签访问JavaBean

1. 声明 JavaBean 对象

<jsp:useBean id="对象名" class="完全限定类名" scope="有效范围" />

scope：page、request、session、application



2. 设置 JavaBean 属性值

（1）简单设置

< jsp:setProperty name="beanname" property="propertyname" value="beanvelue" />

将 beanname 对象的 propertyname 属性设置为 beanvalue，自动实现类型转换


（2）将单个属性和输入参数直接关联

<jsp:setProperty name="beanname" property="propertyname"/>

将参数名称为 propertyname 的值提交给 JavaBean属性名称相同的属性，自动实现类型转换


（3）将单个属性与输入参数间接关联

<jsp:setProperty name="beanname" property="propertyname" param="paramname" />

将名称为 paramname 的请求参数的值给 JavaBean 的 propertyname 属性设置值


（4）将所有的属性与请求参数关联

<jsp:setProperty name="beanname" property="*" />

把提交页面中表单输入域所提供的输入值提交到 JavaBean 中相同名称的属性中



3. 获取 JavaBean 的值并显示

<jsp:getProperty name="beanname" property="propertyname" />

！！jsp：getProperty 动作标签是通过 JavaBean 中的 get 方法获取对应属性的值

各种开发模式

如何理解MVC开发模式

77

在开发一个Web应用程序时，通常需要同时使用这3种技术，并分别承担不同的职责

JSP 一般用来编写用户界面层的信息显示，充当视图层的角色（简称为V）

Serlet 主要用来扮演任务的执行者，一般充当着控制层的角色（简称为C)）

JavaBean 主要实现业务逻辑的处理，充当模型层的角色（简称为M）

实现了不同组件的功能分工协作，将一个系统的功能分为 3 种不同类型的组件，这种模式称为 MVC 模式


JSP+JavaBean

将要进行的业务逻辑封装到 JavaBean 中，在 jsp 页面通过动作标签来调用 JavaBean 类。

JSP 除了负责流程的控制，大部分用来进行页面的显示；JavaBean 则负责业务逻辑的处理


JSP+Servlet

在 JSP+Servlet 开发模式中

JSP 只负责数据的输入（提交请求）和输出（显示请求结果）

而业务逻辑和流程的控制由 Servlet 完成。
