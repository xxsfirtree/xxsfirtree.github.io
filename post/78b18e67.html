JSP

JSP 是一种动态网页技术，主要的作用是 替代 servlet 程序 回传 HTML 页面

JSP 页面主要是由 HTML 和 JSP 代码组成的，JSP 代码通过 <% %> 加入到 HTML 代码中

本质上，jsp 就是一个 servlet 程序


JSP 基本元素

JSP 脚本元素

JSP 脚本元素包括 JSP 声明、JSP 表达式、JSP 代码块，在页面初始化时被初始化

JSP 声明

<%! 声明变量、方法、类 %>

可以只声明变量，也可以在声明的同时提供初始值


JSP 表达式脚本

<%=表达式%>

是由变量、常量组成的算式，将数值转化为字符串嵌入 html 页面，并直接输出

注意事项：

  • 不能用分号结尾
  • 标签内部不能有空格
  • 表达式脚本都会被翻译成为 out.print() 输出到页面上
  • 表达式脚本会被翻译到 _jspService() 方法中，所以该方法中的对象（request, response）都可以直接调用

JSP 代码块

<% 符合 Java 语法的代码块 %>

代码块脚本也会被翻译到 _jspService() 方法中，所以该方法中的对象（request, response）也可以直接调用

代码脚本也可以跟表达式脚本一起组合使用


JSP 指令元素

JSP 指令的语法格式：

<% 指令名称 属性1="属性值1" 属性2="属性值2" ... %>


JSP 指令包括：page 指令、include 指令和 taglib 指令

  • page 指令：定义整个页面的全局属性
  • include 指令：用于包含一个文本或代码的文件
  • taglib 指令：用来引用自定义的标签或第三方标签库

page 指令

用来定义 jsp 页面中的全局属性，描述了与页面相关的一些信息

属性 说明
language 指定用到的脚本语言，默认是 java
import 用于导入 Java包和 Java 类
pageEncoding 指定页面所用编码

注意事项：

  1. 在每个页面中可以使用多个 <%@ page %>指令，分别描述不同的属性
  2. 除了 import，每个属性仅能使用一次
  3. 区分大小写

include 指令

称为文件加载指令，将其他的文件插入 JSP 页面，被插入的文件必须保证插入后形成的新文件符合 JSP 页面的语法规则

<%@ include file="" %>

file 属性指定你要包含的 jsp 页面的路径，地址中第一个斜杠 / 表示为 http://ip:port/工程路径/，映射到代码的 web 目录

  1. 静态包含不会翻译被包含的 jsp 页面
  2. 静态包含其实是把被包含的 jsp 页面的代码拷贝到包含的位置执行输出

JSP 动作元素

用来控制 JSP 引擎的行为，主要有以下六个动作元素

  • <jsp:include>：在页面得到请求时动态包含另一个网页
  • <jsp:forward>：转向
  • <jsp:plugin>：连接客户端的 Applet 或者 Bean 组件
  • <jsp:useBean>：应用 JavaBean 组件
  • <jsp:serProperty>：设置 JavaBean 的属性值
  • <jsp:getProperty>：获取 JavaBean 的属性值并输出

jsp:include

<jsp:include page="" />

动态包含另一个页面

  • 当前页面和被包含的页面是各自独立被翻译的
  • 当执行到该动作标签时，才加载执行被包含文件

jsp:forward

<jsp:forward page="" />

页面的转向：停止当前页面的执行，转向另一个页面，浏览器的地址栏不会跳转


jsp:param

param不能单独使用，而是作为 jsp:include 或 jsp:forward 的子标签使用，用于传递数据

注释

（1）html 注释

<!-- 注释 -->

html 注释会被翻译到 java 源代码中。在 _ jspService 方法里，以 out.writer 输出到客户端


（2）Java 注释

<% // 注释 %>

java 注释会被翻译到 java 源代码中


（3）jsp 注释

<%-- 注释 -->

jsp 注释可以注掉 jsp 页面中所有代码


JSP 内置对象

九大内置对象

对象名称 有效范围 说明
application application 表示应用程序上下文
config page 允许将初始化数据传递给一个 jsp 页面
exception page 异常处理
out page 提供对输出流的访问
page page 代表 jsp 页面对应的 Servlet 类实例
pageContext page 是 jsp 页面本身的上下文
request request 提供对请求数据的访问
response page 用来向客户端输入数据
session session 用来保存在服务器端与一个客户端之间需要保存的数据，当客户端关闭所有网页时自动消失

内置对象的作用域

作用域 说明
page 当前 jsp 页面范围内有效
request 一次请求内有效
session 一个会话范围内有效
application 整个 web 工程范围内都有效

response 和 out 的输出

response.write() 和 out.write() 都会输出内容到客户端

但是以下代码会先输出 “response” 再输出 “out”

原因是，当 jsp 页面中所有代码执行完成后会做以下两个操作：

  1. 执行 out.flush() 操作，会把 out 缓冲区中的数据追加写入到 response 缓冲区末尾
  2. 会执行 response 的刷新操作，把全部数据写给客户端

由于 jsp 翻译后，底层源码都是使用 out 来进行输出，所以我们在 jsp 页面一般统一使用 out，避免打乱页面输出顺序

  • out.write() 输出字符串没有问题，但是输出数值时，会直接转为字符导致错误
  • out.print() 输出任意数据都没有问题（都转换成为字符串后调用的 write 输出）

综上，在 jsp 页面中，可以统一使用 out.print() 来进行输出


response 的重定向

response.sendRedirect("xxx.jsp")

Listener 监听器

JavaWeb 的三大组件之一，是 JavaEE 的规范（接口）

监听器的作用是，监听某种事物的变化。然后通过回调函数，反馈给客户（程序）去做一些相应的处理

目前只有 ServletContextListener 监听器仍在使用


  • ServletContextListener 可以监听 ServletContext 对象的创建和销毁
  • ServletContext 对象在 web 工程启动的时候创建，在 web 工程停止的时候销毁

使用步骤：

  1. 编写一个类实现 ServletContextListener 接口
  2. 实现两个回调方法
  3. 到 web.xml 中配置相关信息

请求转发的流程

  1. 客户端获取关键字，然后传给 Servlet 程序
  2. servlet 的功能
    1. 获取请求的参数
    2. 业务处理
    3. 将数据信息保存到 request 对象中，然后请求转发
  3. jsp 则专门用来显示数据信息
    1. 从 request 中获取数据信息
    2. 输出可用信到客户端，显示

EL 表达式

EL 表达式的全称是：Expression Language，是表达式语言

作用：主要是代替 jsp 页面中的表达式脚本在 jsp 页面中进行数据的输出，因为简洁


表达式脚本输出空值时是 null，而 EL 表达式输出的是空字符串

基本语法：${ 表达式 }


若不指定查找范围，则依次在 page、request、session、application 范围中查找（从小到大）

EL 在输出属性值时，会自动的寻找该属性的get 方法，若没有 get 方法则会产生错误

以上两种写法相同


运算符

算数运算符和逻辑运算符同 java，比较特殊的有

empty 运算符

可以判断一个数据是否为空，以下几种情况为空：

  1. 值为 null 值的时候，为空
  2. 值为空串的时候，为空
  3. 值是 Object 类型数组，长度为零的时候，为空
  4. 集合，元素个数为零的时候，为空

. 运算和 [] 运算

${name, property} 或 ${name["property"]}

  • [] 主要用于访问数组、列表或其他集合对象的属性
  • . 主要用于访问对象的属性
  • [] 和 . 在访问对象属性时可以通用，但当属性名中有特殊字符（, - +）时需要用[]，且要加引号 [“xxx”]

JSTL

是 JSP 标准标签库，可以提高开发效率，减少 JSP 页面中的代码数量

EL 主要是为了替代 表达式脚本，JSTL 主要是为了替代 代码脚本

功能类型 URI prefix 功能
核心库 http://java.sun.com/jsp/jstl/core c 操作范围变量、流程控制、URL 生成和操作
函数 http://java.sun.com/jsp/jstl/functions fn 字符串处理函数
格式化 http://java.sun.com/jsp/jstl/fmt fmt 数字及日期数据格式化
数据库存取（不用） http://java.sun.com/jsp/jstl/sql sql 操作关系数据库
XML 处理（不用） http://java.sun.com/jsp/jstl/xml x 操作通过 XML 表示的数据

在 web 中使用 jstl 的步骤如下：

  1. 导入 Jar 包
  2. 在 JSP 页面中添加 taglib 指令<%@ taglib prefix="" url="" %>
  3. 在页面中使用标签 <c:out value="${1+2}" />

PS：JSTL 通常和 EL 表达式结合使用，EL 作为 JSTL 标签的属性值


常用的 JSTL 标签

<c:out>

用于在 JSP 页面中显示数据

<c:out value="${ user.userName }" default="默认值" />


<c:set>

<c:set target="" value="" var="" property="" />用于保存数据

eg. 将 test.testinfo 的值保存到 session 的 test2 中，其中 test 是一个 JavaBean 实例

<c:set value="${test.testinfo}" var="test2" scope="session" />

将对象 cust.address 的 city 属性保存到变量 city 中

<c:out target="${cust.address}" value="{city}" property="city" />

注意事项：

  • value – 要保存的信息，可以是 EL 表达式或者常量
  • target – 需要修改属性的变量名，一般是 JavaBean 实例
  • property – 需要修改的 JavaBean 属性
  • var – 需要保存信息的变量
  • scope – 保存信息的变量范围

**若指定了 target ，则必须指定 property **


<c:remove>

<c:remove var="" scope="" /> 用于删除数据

  • var – 要删除的变量
  • scope – 被删除变量的范围

<c:if>

  • test – 表示需要判断的条件
  • var – 表示保存条件结果的变量名
  • scope – 保存条件结果的变量范围

<c:choose>

  1. 标签里不能使用 html 注释，要使用 jsp 注释
  2. when 标签的父标签一定要是 choose 标签

  • begin – 表示开始的索引
  • end – 表示结束的索引
  • var – 表示循环的变量
  • step – 表示循环步长

（2）遍历 Object 数组

<c:forEach items="" var=""></c:forEach>

  • items – 表示遍历的集合
  • var – 表示当前遍历到的数据

类似于 for (Object item : items)

文件的上传

1、要有一个 form 标签，method=post 请求

2、form 标签的 encType 属性值必须为 multipart / form-data 值

3、在 form 标签中使用 input type=file，添加上传的文件

4、编写服务器代码接收，处理上传的数据

encType=multipart/ form- data 表示提交的数据以多段的形式进行拼接，然后以二进制流的形式发送给服务器


我们一般使用 commons-fileupload.jar 完成文件的上传

commons-fileupload.jar 需要依赖 commons-io.jar

常见的类和 API

  • ServletFileUpload 类，用于解析上传的数据

  • Fileltem 类，表示每一个表单项

  • boolean ServletFileUpload.isMultipartContent(HttpServletRequest request) 判断当前上传的数据格式是否是多段的格式

  • public List<FileItem> parseRequest(HttpServletRequest request) 解析上传的数据

  • boolean FileItem.isFormField() 判断当前这个表单项，是否是普通的表单项

    • true 表示普通类型的表单项
    • false 表示上传的文件类型

  • String FileItem.getFieldName() 获取表单项的 name 属性值

  • String FileItem.getString() 获取当前表单项的值

  • String FileItem.getName( ) 获取上传的文件名

  • void FileItem.write( file ) 将上传的文件写到参数 file 所指向的硬盘位置

MVC

MVC全称：Model 模型、View 视图、Controller 控制器

View 视图：只负责数据和界面的显示，不接受任何与显示数据无关的代码，便于程序员和美工的分工合作：JSP / HTML

Controller 控制器：只负责接收请求，调用业务层的代码处理请求，然后派发页面，是一个“调度者”的角色：Servlet

Model 模型：将与业务逻辑相关的数据封装为具体的 JavaBean 类，其中不掺杂任何与数据处理相关的代码：JavaBean

MVC是一种思想，理念是将软件代码拆分成为组件，单独开发，组合使用（目的还是为了解耦合）


一个项目工程一般都分为前台和后台

  • 前台是给普通用户使用，一般不需要权限检查， 就可以访问的资源
  • 后台是给管理员使用，一般都需要权限检查，才可以访问的资源 / 页面

我们通过不同的路径区分前台和后台

  • 前台 /client / xxxServlet
  • 后台 /manager/xxxServlet
