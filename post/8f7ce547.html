提交表单时不能正确的把信息发给服务器的原因：

  • 表单没有设置 name 属性
  • 单选、复选、下拉框等标签都需要添加 value 标签，以便向服务器传递信息
  • 表单项不在提交的 form 标签之中

methon 属性确定提交方式：

一般是 post 和 get（默认）两种

get 和 post 两种提交方式

get 请求的特点是：

  1. 格式固定：name=value & name=value…

  2. 不安全，会显示在浏览器地址栏中

  3. 对数据的长度有限制，可以提交的数据量非常小


post 请求的特点是：

  1. 安全，不会显示在浏览器的地址栏中

  2. 可以提交的数据量非常大，理论上没有限制

  3. 可以进行文件的提交

JavaScript

是一种简单的脚本语言，可以在浏览器中直接运行，是一种在浏览器端实现网页与客户交互的技术

特点：交互性、安全性、跨平台性

基本语法

数据类型：

主要的数据类型有：int、float、string、boolean

undefined：未定义，所有 js 变量未赋初始值时，默认都是该类型

null：空值

NAN：非数字


变量：

使用 var 定义变量，声明变量时不需要指定变量的类型，赋值时自动确定

在 js 中，所有的变量都可以作为 Boolean类型使用

0、null、“”、undefined、NaN 都认为是 false


1）变量声明

var a[ = 10];

2）数组声明

js 中的数组，只要通过下标赋值，那么会自动根据最大下标扩容，没有赋值的位置默认为 undefined


运算符

基本与 java 相同，值得一说的是：

==（等于）：仅仅是字面值的比较，’12‘ 和 12 相等

=== （全等于）：在数值相同的基础上，类型也要相同，’12‘ 和 12 不同


对于 && 与运算：

  • 当表达式全真时，返回最后一个表达式的值
  • 当表达式有一个为假时，返回第一个为假的表达式的值

对于 || 或运算：

  • 当表达式全假时，返回最后一个表达式的值
  • 当表达式有一个为真时，返回第一个为真的表达式的值

PS：&& 和 || 都有短路，即有了运算结果后，后面的表达式不再执行

控制语句

分支语句（if、switch）、循环语句（while、for），规则都跟 java 一样


函数的定义和使用

在 JS 中，需要先声明定义，然后再调用函数

  • 函数的参数列表不需要数据类型，var 也不需要，实质上仅确定了参数数量
  • 函数若有返回值，直接 return 即可
  • 函数不存在重载，会覆盖掉上一次的定义
  • 函数存在 argument 隐形参数

隐形参数：

即使在 function 函数中没有定义，也可以直接获取所有参数的变量，就像 java 里的可变长参数一样


定义函数一般两种方式：

（1）直接在 Web 页面中嵌入

（2）链接外部 JavaScript

将脚本代码保存在一个单独的文件中，其扩展名为.js，然后在 Web 页面链接

（3）函数调用

一般是由事件引起的


PS：<script></script>既可以定义js代码，又可以引入js文件，两者只能选其一，写在一起没用。可以分开写

操作元素

attr() / prop() 方法

  • attr(name) 获取属性的值
  • attr(key, value)
  • attr({key0:value0, key1:value1})
  • attr(key, funvtion() ) 用函数的返回值作为元素的属性

prop() 和 attr() 都是用来获取和设置属性的值

区别：

  • 在选中情况下，prop(checked) 返回 true，attr(checked) 返回checked
  • 在未选中情况下，prop(checked) 返回 false，prop(checked) 返回 undefined

建议：

attr() – 不推荐操作 checked、readOnly、selected、disabled 等，但该方法可以操作非标准属性，如自定义属性

prop() – 仅推荐操作 checked、readOnly、selected、disabled 等

  2. 第二个参数是动画的回调函数（动画完成后自动调用的函数）

淡化动画

  • fadeIn – 淡入
  • fadeOut – 淡出
  • fadeTo(时间，透明度) – 在指定时长内慢慢的将透明度修改到指定的值
  • fadeToggle – 淡入/淡出切换

JQuery 事件

对于页面加载完成之后

  • jQuery 会较 原生js 先执行
  • jQuery 是浏览器的内核解析完页面的标签创建好 DOM 对象之后就马上执行
  • 原生 js 还要等标签显示时需要的内容加载
  • 原生js的页面加载完成之后，只会执行最后一次的赋值函数
  • jQuery的页面加载完成之后是全部把注册的function函数，依次顺序全部执行

事件的冒泡

是指父子元素同时监听同一个事件。

当触发子元素的事件的时候，同一个事件也被传递到了父元素的事件里去响应

那么如何阻止事件冒泡呢?

在子元素事件函数体内 return false; 可以阻止事件的冒泡传递


javaScript 事件对象

事件对象，是封装有触发的事件信息的一个javascript对象

如何获取呢 javascript 事件对象呢?

在给元素绑定事件的时候，在事件的 function( event ) 参数列表中添加一个参数，这个参数名我们习惯为event，这个 event 就是 javascript 传递参事件处理函数的事件对象
